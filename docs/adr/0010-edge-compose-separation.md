# ADR-0010: 리버스 프록시(Caddy)는 앱 Compose에서 분리된 엣지 계층

- 날짜: 2026-07-02 / 상태: Accepted

## 맥락

Caddy를 앱 compose에 포함하면 앱 재배포가 TLS 종단·인증서 상태(ACME 계정)를 함께 재시작하고, :80/:443을 앱이 점유해 같은 인스턴스에 두 번째 프로젝트 추가 시 즉시 재구조화가 필요하며, 잘못된 배포가 진입점까지 끌어내린다 — 수명주기 불일치.

## 검토한 대안

1. 앱 compose에 포함: 위 수명주기 문제 — 기각
2. 호스트 설치(apt+systemd): 단순하나 설정이 버전 관리 밖에서 표류
3. **별도 인프라 compose + 외부 Docker 네트워크(edge)** (채택)

## 결정

Caddy는 자체 compose 프로젝트(엣지 계층)로 :80/:443과 인증서 볼륨을 소유하고 외부 네트워크 `edge`를 제공, 앱 compose는 그 네트워크에 참여만 한다. 새 프로젝트 추가 = Caddyfile 라우트 추가.

## 결과

- k3s 2단계 전환(ADR-0001) 시 엣지=ingress / 앱=workload 대응이 그대로 성립.
- 구현 시점: M5(배포). 로컬 개발 compose에는 엣지 불요.
