# 10. 품질 요구 (시나리오)

품질 목표(§1)의 검증 가능한 시나리오. M1~M4에서 통합 테스트로 고정한다.

| # | 자극 | 기대 반응 | 품질 | 검증 |
|---|---|---|---|---|
| Q1 | 같은 피드로 2회 연속 실행 | 대상 앱 무변화 (create 0, update 0) | 멱등성 | 통합 테스트 |
| Q2 | 과제 마감일 변경 후 실행 | 해당 항목 update 1건만 | 멱등성 | 통합 테스트 |
| Q3 | 사용자가 대상에서 완료 처리 후 소스 변경 도착 | 불개입 (FROZEN 유지) | 사용자 의사 존중 | 통합 테스트 |
| Q4 | 사용자가 대상에서 항목 삭제 | 재생성하지 않음 (FROZEN/DELETED) | 사용자 의사 존중 | 통합 테스트 |
| Q5 | 구독 A의 피드가 타임아웃 | 구독 B 실행 무영향, A는 3연속 실패 시 ERROR+알림 | 장애 격리 | 통합 테스트 |
| Q6 | Discord webhook 죽음 | 실행 결과 무영향, 이벤트 재전송(at-least-once) | 장애 격리 | 통합 테스트 |
| Q7 | 앱 재시작(재배포) | 미완료 통지 이벤트 재전송, 세션 유지(M2+) | 복구 | 수동+테스트 |
| Q8 | 피드에 새 과목 등장 (미매핑 slot) | `_inbox`로 유입 + 대시보드 알림 — 누락 0 | 정확성 | 통합 테스트 |
| Q9 | 새 대상(예: Todoist) 추가 | 기존 모듈 무수정 — delivery 어댑터 추가로 완결 | 확장 개방 | Modulith 검증 |
| Q10 | 잘못된 정규식(백트래킹 폭발) 유입 | 매칭 100ms 타임아웃, 실행은 계속 | 격리 | 단위 테스트 |
